## A) Produktdefinition

### A.1 Vision

> **Schadenradar** ist das zentrale Operations-Dashboard für Multiplikatoren im Schadenmanagement. Es ersetzt fragmentierte Kommunikation (E-Mail, Telefon, Trello) durch eine einheitliche, kartenbasierte Echtzeitübersicht aller Schadenfälle mit automatischer Dokumentenverarbeitung.

### A.2 Strategische Ziele

| Ziel | Metriken | Zeithorizont |
|------|----------|--------------|
| **Transparenz** | 100% Echtzeit-Statusvisibilität für Multiplikatoren | MVP |
| **Effizienz** | -50% manuelle Dokumenten-Zuordnung durch OCR-Automation | Phase 1 |
| **Single Source of Truth** | 0% Datenredundanz durch ERP-First-Architektur | Phase 2 |
| **Self-Service** | 80% Self-Service-Quote für Status-Abfragen | MVP |
| **Compliance** | 100% Audit-Trail-Abdeckung | MVP |

### A.3 Non-Goals (explizit ausgeschlossen)

- ❌ **Keine ERP-Ersatzfunktion** – Schadenradar ist Viewer/Kollaborationsschicht, nicht Datenhaltung
- ❌ **Keine Finanzbuchhaltung** – Angebote/Rechnungen werden angezeigt, nicht verwaltet
- ❌ **Keine Terminplanung** – Kalender ist read-only Sync vom ERP
- ❌ **Kein Dark Mode** (MVP) – Corporate Design ist Light Mode
- ❌ **Keine native Mobile App** (MVP) – PWA first
- ❌ **Keine Multi-Language** (MVP) – Deutsch only

### B.4 Row-Level Security Policies (Supabase)

```sql
-- Tenant Isolation Policy für Claims
CREATE POLICY "tenant_isolation_claims" ON claims
    FOR ALL
    USING (
        tenant_id = auth.jwt() ->> 'tenant_id'
        OR 
        EXISTS (
            SELECT 1 FROM claim_assignments ca
            WHERE ca.claim_id = claims.id 
            AND ca.user_id = auth.uid()
        )
        OR
        auth.jwt() ->> 'role' = 'SYSTEM_ADMIN'
    );

-- Reviewer Cross-Tenant Access
CREATE POLICY "reviewer_access_claims" ON claims
    FOR SELECT
    USING (
        EXISTS (
            SELECT 1 FROM claim_reviewers cr
            WHERE cr.claim_id = claims.id
            AND cr.user_id = auth.uid()
            AND cr.active = true
        )
    );

-- Document Access follows Claim Access
CREATE POLICY "doc_follows_claim" ON documents
    FOR ALL
    USING (
        EXISTS (
            SELECT 1 FROM claims c
            WHERE c.id = documents.claim_id
            AND (
                c.tenant_id = auth.jwt() ->> 'tenant_id'
                OR auth.jwt() ->> 'role' = 'SYSTEM_ADMIN'
            )
        )
    );
```

#### Ampel-Berechnung (Pseudocode)

```typescript
// services/traffic-light.service.ts

interface TrafficLightResult {
  color: 'GREEN' | 'YELLOW' | 'RED';
  reason: string;
  daysOverdue?: number;
  deadline?: Date;
}

function calculateTrafficLight(
  claim: Claim,
  phase: Phase,
  now: Date = new Date()
): TrafficLightResult {
  
  // 1. Check: Blocked Status (always RED)
  if (claim.flags?.blocked) {
    return { color: 'RED', reason: 'BLOCKED_FLAG' };
  }
  
  // 2. Check: Phase SLA
  const phaseAge = businessDaysDiff(phase.started_at, now);
  const phaseSLA = PHASE_SLA[phase.phase_type];
  
  if (phaseAge > phaseSLA.red_threshold) {
    return {
      color: 'RED',
      reason: 'PHASE_OVERDUE',
      daysOverdue: phaseAge - phaseSLA.red_threshold
    };
  }
  
  if (phaseAge > phaseSLA.yellow_threshold) {
    return {
      color: 'YELLOW',
      reason: 'PHASE_AT_RISK',
      deadline: addBusinessDays(phase.started_at, phaseSLA.red_threshold)
    };
  }
  
  // 3. Check: Status SLA
  const statusAge = hoursDiff(phase.status_changed_at, now);
  const statusSLA = STATUS_SLA[phase.status];
  
  if (statusSLA && statusAge > statusSLA.red_hours) {
    return {
      color: 'RED',
      reason: 'STATUS_OVERDUE',
      daysOverdue: Math.ceil((statusAge - statusSLA.red_hours) / 24)
    };
  }
  
  if (statusSLA && statusAge > statusSLA.yellow_hours) {
    return {
      color: 'YELLOW',
      reason: 'STATUS_AT_RISK'
    };
  }
  
  // 4. Check: Appointment overdue
  const nextAppointment = getNextAppointment(claim.id, phase.id);
  if (nextAppointment && nextAppointment.scheduled_start < now 
      && nextAppointment.status !== 'COMPLETED') {
    return {
      color: 'RED',
      reason: 'APPOINTMENT_OVERDUE'
    };
  }
  
  // 5. Check: Priority escalation
  if (claim.priority === 'CRITICAL') {
    // Kritische Schäden haben engere Thresholds
    if (phaseAge > phaseSLA.yellow_threshold * 0.5) {
      return { color: 'YELLOW', reason: 'CRITICAL_PRIORITY' };
    }
  }
  
  return { color: 'GREEN', reason: 'ON_TRACK' };
}

// Aggregate Traffic Light für Claim (alle aktiven Phasen)
function calculateClaimTrafficLight(claim: Claim): TrafficLightResult {
  const activePhases = claim.phases.filter(p => 
    p.status !== PortalStatus.ABGESCHLOSSEN
  );
  
  const phaseTrafficLights = activePhases.map(p => 
    calculateTrafficLight(claim, p)
  );
  
  // Worst-case wins
  if (phaseTrafficLights.some(t => t.color === 'RED')) {
    return phaseTrafficLights.find(t => t.color === 'RED')!;
  }
  if (phaseTrafficLights.some(t => t.color === 'YELLOW')) {
    return phaseTrafficLights.find(t => t.color === 'YELLOW')!;
  }
  
  return { color: 'GREEN', reason: 'ALL_ON_TRACK' };
}
```

### E.5 Versioning

```sql
-- Document versioning via parent reference
-- v1: parent_version_id = NULL, version = 1
-- v2: parent_version_id = v1.id, version = 2

-- Get latest version
CREATE VIEW documents_latest AS
SELECT DISTINCT ON (COALESCE(parent_version_id, id)) *
FROM documents
ORDER BY COALESCE(parent_version_id, id), version DESC;

-- Version history for a document
CREATE OR REPLACE FUNCTION get_document_versions(doc_id UUID)
RETURNS TABLE (
    id UUID,
    version INTEGER,
    filename VARCHAR,
    uploaded_at TIMESTAMP,
    uploaded_by UUID
) AS $$
    WITH RECURSIVE version_chain AS (
        SELECT id, version, filename, uploaded_at, uploaded_by, parent_version_id
        FROM documents WHERE id = doc_id
        UNION ALL
        SELECT d.id, d.version, d.filename, d.uploaded_at, d.uploaded_by, d.parent_version_id
        FROM documents d
        INNER JOIN version_chain vc ON d.id = vc.parent_version_id
    )
    SELECT id, version, filename, uploaded_at, uploaded_by
    FROM version_chain
    ORDER BY version DESC;
$$ LANGUAGE sql;
```

```tsx
import { MapContainer, TileLayer, Marker, Popup } from 'react-leaflet';
import MarkerClusterGroup from 'react-leaflet-cluster';
import L from 'leaflet';

function RadarMap({ claims }: { claims: ClaimWithTrafficLight[] }) {
  const createClusterCustomIcon = (cluster: L.MarkerCluster) => {
    const childMarkers = cluster.getAllChildMarkers();
    const hasRed = childMarkers.some(m => m.options.trafficLight === 'RED');
    const hasYellow = childMarkers.some(m => m.options.trafficLight === 'YELLOW');
    
    const color = hasRed ? 'bg-status-red' : hasYellow ? 'bg-status-yellow' : 'bg-status-green';
    
    return L.divIcon({
      html: `<div class="${color} w-8 h-8 rounded-full flex items-center justify-center text-white font-bold">
               ${cluster.getChildCount()}
             </div>`,
      className: 'custom-cluster-icon',
      iconSize: L.point(32, 32),
    });
  };

  return (
    <MapContainer
      center={[51.1657, 10.4515]} // Germany center
      zoom={6}
      className="h-[calc(100vh-200px)] rounded-card"
    >
      <TileLayer
        url="https://{s}.tile.openstreetmap.org/{z}/{x}/{y}.png"
        attribution='&copy; OpenStreetMap'
      />
      
      <MarkerClusterGroup
        chunkedLoading
        iconCreateFunction={createClusterCustomIcon}
      >
        {claims.map(claim => (
          <ClaimMarker key={claim.id} claim={claim} />
        ))}
      </MarkerClusterGroup>
    </MapContainer>
  );
}

function ClaimMarker({ claim }: { claim: ClaimWithTrafficLight }) {
  const icon = createTrafficLightIcon(claim.trafficLight.color);
  
  return (

      icon={icon}
      trafficLight={claim.trafficLight.color}
    >
      <Popup>
        <ClaimQuickPreview claim={claim} />
      </Popup>
    </Marker>
  );
}

function createTrafficLightIcon(color: TrafficLight): L.DivIcon {
  const colorClass = {
    GREEN: 'bg-status-green',

  return L.divIcon({
    html: `<div class="${colorClass} w-4 h-4 rounded-full border-2 border-white shadow-lg"></div>`,
    className: 'custom-marker-icon',
    iconSize: L.point(16, 16),
    iconAnchor: L.point(8, 8),
  });
}
```

```yaml
# n8n Workflow: erp_status_sync

trigger:
  type: webhook
  path: /erp/status-changed
  method: POST

nodes:
  - name: Validate Payload
    type: function
    code: |
      const { erp_project_number, erp_substatus, timestamp, event_id } = $input.body;
      
      if (!erp_project_number || !erp_substatus) {
        throw new Error('Invalid payload');
      }
      
      return {
        json: {
          erpProjectNumber: erp_project_number,
          erpSubstatus: erp_substatus,
          timestamp: timestamp,
          eventId: event_id,
          idempotencyKey: `status_${erp_project_number}_${event_id}`
        }
      };

  - name: Check Idempotency
    type: supabase
    operation: select
    table: erp_sync_log
    filters:
      - field: payload_hash
        operator: eq
        value: "={{ $json.idempotencyKey }}"
    onNoResults: continue
    onResults: stop  # Already processed

  - name: Map to Portal Status
    type: function
    code: |
      const mapping = {
        'ORTSTERMIN_OFFEN': 'TERMIN_OFFEN',
        'ORTSTERMIN_BEAUFTRAGT': 'TERMIN_GEPLANT',
        // ... full mapping
      };
      
      return {
        json: {
          ...$json,
          portalStatus: mapping[$json.erpSubstatus] || 'UNKNOWN'
        }
      };

  - name: Update Claim Phase
    type: supabase
    operation: update
    table: phases
    filters:
      - field: claim.erp_project_number
        operator: eq
        value: "={{ $json.erpProjectNumber }}"
    data:
      status: "={{ $json.portalStatus }}"
      erp_substatus: "={{ $json.erpSubstatus }}"
      status_changed_at: "={{ $json.timestamp }}"

  - name: Create Timeline Event
    type: supabase
    operation: insert
    table: timeline_events
    data:
      claim_id: "={{ $node['Update Claim Phase'].data.claim_id }}"
      event_type: STATUS_CHANGE
      title: "Status geändert: {{ $json.portalStatus }}"
      metadata:
        from_erp: true
        erp_substatus: "={{ $json.erpSubstatus }}"
      erp_event_id: "={{ $json.eventId }}"

  - name: Log Sync
    type: supabase
    operation: insert
    table: erp_sync_log
    data:
      entity_type: status
      erp_entity_id: "={{ $json.eventId }}"
      direction: ERP_TO_PORTAL
      operation: UPDATE
      payload_hash: "={{ $json.idempotencyKey }}"
      status: SUCCESS
```

```yaml
# n8n Workflow: portal_document_to_erp

trigger:
  type: supabase_realtime
  table: documents
  event: INSERT

nodes:

    type: function
```

### G.4 Conflict Resolution

```typescript
// services/conflict-resolver.ts

enum ConflictStrategy {
  ERP_WINS = 'ERP_WINS',           // Default for status/master data
  PORTAL_WINS = 'PORTAL_WINS',     // For portal-native data
  MERGE = 'MERGE',                 // For comments/timeline
  MANUAL = 'MANUAL',               // Flag for human review
}

interface ConflictResolution {
  entityType: string;
  strategy: ConflictStrategy;
  mergeLogic?: (erpData: any, portalData: any) => any;
}

const conflictStrategies: Record<string, ConflictResolution> = {
  'claim.status': {
    entityType: 'claim.status',
    strategy: ConflictStrategy.ERP_WINS,
  },
  'claim.contacts': {
    entityType: 'claim.contacts',
    strategy: ConflictStrategy.ERP_WINS,
  },
  'document.metadata': {
    entityType: 'document.metadata',
    strategy: ConflictStrategy.MERGE,
    mergeLogic: (erp, portal) => ({
      ...erp,
      portal_tags: portal.portal_tags,
      portal_notes: portal.portal_notes,
    }),
  },
  'timeline_event': {
    entityType: 'timeline_event',
    strategy: ConflictStrategy.MERGE,
    mergeLogic: (erp, portal) => {
      // Dedupe by erp_event_id
      const merged = [...erp];
      portal.forEach(e => {
        if (!e.erp_event_id || !merged.some(m => m.erp_event_id === e.erp_event_id)) {
          merged.push(e);
        }
      });
      return merged.sort((a, b) => b.created_at - a.created_at);
    },
  },
};
```

```typescript
    redactFields: ['erp_substatus', 'internal_notes'],
  },
  {
    eventType: 'INTERNAL_NOTE',
    visibleToExternal: false,
  },
  {
    eventType: 'COST_UPDATE',
    visibleToExternal: false,
  },
  {
    eventType: 'DOCUMENT_UPLOAD',
    visibleToExternal: true,
    redactFields: ['internal_classification'],
  },
  {
    eventType: 'APPOINTMENT',
    visibleToExternal: true,
    redactFields: ['technician_notes', 'resource_allocation'],
  },
];

function censorTimeline(
  events: TimelineEvent[],
  userRole: UserRole
): TimelineEvent[] {
  if (isInternalUser(userRole)) {
    return events;
  }
  
  return events
    .filter(e => {
      const rule = censorRules.find(r => r.eventType === e.event_type);
      return rule?.visibleToExternal !== false;
    })
    .map(e => {
      const rule = censorRules.find(r => r.eventType === e.event_type);
      if (!rule?.redactFields) return e;
      
      const censored = { ...e };
      rule.redactFields.forEach(field => {
        if (censored.metadata?.[field]) {
          delete censored.metadata[field];
        }
      });
      
      if (rule.transformTitle) {
        censored.title = rule.transformTitle(censored.title);
      }
      
      return censored;
    });
}
```

```typescript
// services/ocr/azure-doc-intelligence.ts

import { DocumentAnalysisClient, AzureKeyCredential } from '@azure/ai-form-recognizer';

const client = new DocumentAnalysisClient(
  process.env.AZURE_DI_ENDPOINT!,
  new AzureKeyCredential(process.env.AZURE_DI_KEY!)

interface ExtractionResult {
  rawText: string;
  extractedFields: ExtractedFields;
  confidenceScores: Record<string, number>;
  documentType: DocumentType;
  documentTypeConfidence: number;
  processingTimeMs: number;
  azureOperationId: string;
}

interface ExtractedFields {
  projectNumber?: string;
  claimNumbers?: {
    insurer?: string;
    regulator?: string;
    expert?: string;
  };
  address?: {
    street?: string;
    zipCode?: string;
    city?: string;
  };
  date?: string;
  amount?: number;
  recipient?: string;
  sender?: string;
  persons?: string[];
}

export async function analyzeDocument(
  documentUrl: string,
  mimeType: string
): Promise<ExtractionResult> {
  const startTime = Date.now();
  
  // Use prebuilt-document for general extraction
  // Or custom model for domain-specific docs
  const modelId = selectModel(mimeType);
  
  const poller = await client.beginAnalyzeDocument(modelId, documentUrl);
  const result = await poller.pollUntilDone();
  
  return {
    rawText: extractFullText(result),
    extractedFields: extractFields(result),
    confidenceScores: buildConfidenceMap(result),
    documentType: classifyDocumentType(result),
    documentTypeConfidence: result.documents?.[0]?.confidence || 0,
    processingTimeMs: Date.now() - startTime,
    azureOperationId: result.modelId,
  };
}

function selectModel(mimeType: string): string {
  // Custom models for specific document types
  const customModels: Record<string, string> = {
    'application/pdf': 'prebuilt-document',
    'image/jpeg': 'prebuilt-read',
    // Add custom trained models here
    // 'LO_PROTOKOLL': 'custom-lo-protokoll-v1',
  };
  
  return customModels[mimeType] || 'prebuilt-document';
}

function extractFields(result: AnalyzeResult): ExtractedFields {
  const fields: ExtractedFields = {};
  
  // Project Number Pattern: P followed by 6 digits
  const projectMatch = result.content.match(/P\d{6}/);
  if (projectMatch) {
    fields.projectNumber = projectMatch[0];
  }
  
  // Claim Number Patterns (various formats)
  const claimPatterns = [
    /Schaden(?:nummer|nr\.?)[\s:]*([A-Z0-9\-\/]+)/i,
    /VS[\-\/]?\d{4}[\-\/]?\d+/,
    /SV[\-\/]?\d+[\-\/]?\d*/,
  ];
  
  for (const pattern of claimPatterns) {
    const match = result.content.match(pattern);
    if (match) {
      // Classify which type based on prefix
      if (match[0].startsWith('VS')) {
        fields.claimNumbers = { ...fields.claimNumbers, insurer: match[0] };
      } else if (match[0].startsWith('SV')) {
        fields.claimNumbers = { ...fields.claimNumbers, expert: match[0] };
      }
    }
  }
  
  // Address extraction via key-value pairs
  const addressKv = result.keyValuePairs?.find(
    kv => kv.key?.content?.toLowerCase().includes('adresse')
  );
  if (addressKv?.value?.content) {
    fields.address = parseAddress(addressKv.value.content);
  }
  
  // Amount extraction
  const amountPatterns = [
    /(?:Gesamt|Summe|Betrag)[\s:]*€?\s*([\d.,]+)/i,
    /€\s*([\d.,]+)/,
  ];
  for (const pattern of amountPatterns) {
    const match = result.content.match(pattern);
    if (match) {
      fields.amount = parseGermanNumber(match[1]);
      break;
    }
  }
  
  // Date extraction
  const dateMatch = result.content.match(/\d{1,2}\.\d{1,2}\.\d{2,4}/);
  if (dateMatch) {
    fields.date = dateMatch[0];
  }
  
  // Person names (from entities)
  fields.persons = result.entities
    ?.filter(e => e.category === 'Person')
    .map(e => e.content) || [];
  
  return fields;
}

function classifyDocumentType(result: AnalyzeResult): DocumentType {
  const content = result.content.toLowerCase();
  
  const typePatterns: Array<[DocumentType, RegExp[]]> = [
    [DocumentType.LO_PROTOKOLL, [/leckortung/, /leckageortung/, /lo[\-\s]protokoll/]],
    [DocumentType.LB_PROTOKOLL, [/leckagebehebung/, /lb[\-\s]protokoll/]],
    [DocumentType.ANGEBOT, [/angebot/, /kostenvoranschlag/, /kva/]],
    [DocumentType.RECHNUNG, [/rechnung/, /invoice/]],
    [DocumentType.GUTACHTEN, [/gutachten/, /sachverständig/]],
    [DocumentType.TROCKNUNGSPROTOKOLL, [/trocknungsprotokoll/, /messwerte/]],
    [DocumentType.ABNAHMEPROTOKOLL, [/abnahme/, /übergabe/]],
    [DocumentType.VOLLMACHT, [/vollmacht/, /bevollmächtigung/]],
    [DocumentType.SCHADENMELDUNG, [/schadenmeldung/, /schadenanzei/]],
  ];
  
  for (const [type, patterns] of typePatterns) {
    if (patterns.some(p => p.test(content))) {
      return type;
    }
  }
  
  return DocumentType.SONSTIGES;
}
```

### H.3 Matching Engine

```typescript
// services/ocr/matching-engine.ts

interface MatchCandidate {
  claimId: string;
  score: number;
  matchReasons: MatchReason[];
}

interface MatchReason {
  criterion: MatchCriterion;
  extractedValue: string;
  matchedValue: string;
  score: number;
}

enum MatchCriterion {
  PROJECT_NUMBER = 'PROJECT_NUMBER',
  INSURER_CLAIM_NUMBER = 'INSURER_CLAIM_NUMBER',
  REGULATOR_CLAIM_NUMBER = 'REGULATOR_CLAIM_NUMBER',
  EXPERT_CLAIM_NUMBER = 'EXPERT_CLAIM_NUMBER',
  ADDRESS_EXACT = 'ADDRESS_EXACT',
  ADDRESS_FUZZY = 'ADDRESS_FUZZY',
  PERSON_NAME = 'PERSON_NAME',
  TENANT_CONTEXT = 'TENANT_CONTEXT',
}

// Criterion Weights
const CRITERION_WEIGHTS: Record<MatchCriterion, number> = {
  [MatchCriterion.PROJECT_NUMBER]: 1.0,        // Highest priority
  [MatchCriterion.INSURER_CLAIM_NUMBER]: 0.9,
  [MatchCriterion.REGULATOR_CLAIM_NUMBER]: 0.9,
  [MatchCriterion.EXPERT_CLAIM_NUMBER]: 0.85,
  [MatchCriterion.ADDRESS_EXACT]: 0.8,
  [MatchCriterion.ADDRESS_FUZZY]: 0.5,
  [MatchCriterion.PERSON_NAME]: 0.4,
  [MatchCriterion.TENANT_CONTEXT]: 0.2,        // Boost factor
};

// Confidence Thresholds
const THRESHOLDS = {
  AUTO_ASSIGN: 0.85,
  REVIEW_QUEUE: 0.6,
  UNASSIGNED: 0.0,
};

export async function findMatchingClaim(
  extraction: ExtractionResult,
  tenantId: string
): Promise<{
  bestMatch: MatchCandidate | null;
  alternatives: MatchCandidate[];
  assignmentType: 'AUTO' | 'REVIEW' | 'UNASSIGNED';
}> {
  // Get candidate claims for this tenant
  const candidateClaims = await getCandidateClaims(tenantId);
  const candidates: MatchCandidate[] = [];
  
  for (const claim of candidateClaims) {
    const matchResult = calculateMatchScore(extraction, claim);
    if (matchResult.score > 0) {
      candidates.push({
        claimId: claim.id,
        score: matchResult.score,
        matchReasons: matchResult.reasons,
      });
    }
  }
  
  // Sort by score descending
  candidates.sort((a, b) => b.score - a.score);
  
  const bestMatch = candidates[0] || null;
  const alternatives = candidates.slice(1, 5); // Top 5 alternatives
  
  let assignmentType: 'AUTO' | 'REVIEW' | 'UNASSIGNED';
  if (bestMatch && bestMatch.score >= THRESHOLDS.AUTO_ASSIGN) {
    assignmentType = 'AUTO';
  } else if (bestMatch && bestMatch.score >= THRESHOLDS.REVIEW_QUEUE) {
    assignmentType = 'REVIEW';
  } else {
    assignmentType = 'UNASSIGNED';
  }
  
  return { bestMatch, alternatives, assignmentType };
}

function calculateMatchScore(
  extraction: ExtractionResult,
  claim: Claim
): { score: number; reasons: MatchReason[] } {
  const reasons: MatchReason[] = [];
  
  // 1. Project Number Match (exact)
  if (extraction.extractedFields.projectNumber) {
    if (extraction.extractedFields.projectNumber === claim.erpProjectNumber) {
      reasons.push({
        criterion: MatchCriterion.PROJECT_NUMBER,
        extractedValue: extraction.extractedFields.projectNumber,
        matchedValue: claim.erpProjectNumber,
        score: CRITERION_WEIGHTS[MatchCriterion.PROJECT_NUMBER],
      });
    }
  }
  
  // 2. Claim Numbers Match
  const claimNumbers = extraction.extractedFields.claimNumbers;
  if (claimNumbers?.insurer && 
      claim.externalClaimNumbers?.insurer === claimNumbers.insurer) {
    reasons.push({
      criterion: MatchCriterion.INSURER_CLAIM_NUMBER,
      extractedValue: claimNumbers.insurer,
      matchedValue: claim.externalClaimNumbers.insurer,
      score: CRITERION_WEIGHTS[MatchCriterion.INSURER_CLAIM_NUMBER],
    });
  }
  
  // 3. Address Match
  if (extraction.extractedFields.address) {
    const addressScore = compareAddresses(
      extraction.extractedFields.address,
      claim.property
    );
    
    if (addressScore >= 0.95) {
      reasons.push({
        criterion: MatchCriterion.ADDRESS_EXACT,
        extractedValue: formatAddress(extraction.extractedFields.address),
        matchedValue: formatAddress(claim.property),
        score: CRITERION_WEIGHTS[MatchCriterion.ADDRESS_EXACT],
      });
    } else if (addressScore >= 0.7) {
      reasons.push({
        criterion: MatchCriterion.ADDRESS_FUZZY,
        extractedValue: formatAddress(extraction.extractedFields.address),
        matchedValue: formatAddress(claim.property),
        score: CRITERION_WEIGHTS[MatchCriterion.ADDRESS_FUZZY] * addressScore,
      });
    }
  }
  
  // 4. Person Name Match
  if (extraction.extractedFields.persons?.length) {
    const claimPersons = claim.contacts.map(c => c.name.toLowerCase());
    for (const person of extraction.extractedFields.persons) {
      const matchedPerson = claimPersons.find(
        cp => fuzzyMatch(cp, person.toLowerCase()) >= 0.85
      );
      if (matchedPerson) {
        reasons.push({
          criterion: MatchCriterion.PERSON_NAME,
          extractedValue: person,
          matchedValue: matchedPerson,
          score: CRITERION_WEIGHTS[MatchCriterion.PERSON_NAME],
        });
        break; // One match is enough
      }
    }
  }
  
  // Calculate final score (take best match, not sum)
  // This prevents low-confidence matches from stacking up
  const finalScore = reasons.length > 0
    ? Math.max(...reasons.map(r => r.score))
    : 0;
  
  return { score: finalScore, reasons };
}

function compareAddresses(
  extracted: { street?: string; zipCode?: string; city?: string },
  property: Property
): number {
  let score = 0;
  let weights = 0;
  
  // ZIP code is highly reliable
  if (extracted.zipCode) {
    weights += 0.4;
    if (extracted.zipCode === property.zip) {
      score += 0.4;
    }
  }
  
  // Street name with fuzzy match
  if (extracted.street && property.address) {
    weights += 0.4;
    const streetScore = fuzzyMatch(
      normalizeStreet(extracted.street),
      normalizeStreet(property.address)
    );
    score += 0.4 * streetScore;
  }
  
  // City
  if (extracted.city && property.city) {
    weights += 0.2;
    if (extracted.city.toLowerCase() === property.city.toLowerCase()) {
      score += 0.2;
    }
  }
  
  return weights > 0 ? score / weights : 0;
}

function fuzzyMatch(a: string, b: string): number {
  // Levenshtein distance normalized to 0-1
  const distance = levenshtein(a, b);
  const maxLength = Math.max(a.length, b.length);
  return 1 - (distance / maxLength);
}
```

### H.4 Processing Pipeline (Supabase Edge Function)

```typescript
// supabase/functions/process-document/index.ts

import { serve } from 'https://deno.land/std@0.177.0/http/server.ts';
import { createClient } from '@supabase/supabase-js';

serve(async (req) => {
  const supabase = createClient(
    Deno.env.get('SUPABASE_URL')!,
    Deno.env.get('SUPABASE_SERVICE_ROLE_KEY')!
  );
  
  const { documentId } = await req.json();
  
  // 1. Get document
  const { data: doc } = await supabase
    .from('documents')
    .select('*')
    .eq('id', documentId)
    .single();
  
  if (!doc) {
    return new Response(JSON.stringify({ error: 'Document not found' }), {
      status: 404,
    });
  }
  
  // 2. Update status to PROCESSING
  await supabase
    .from('documents')
    .update({ ocr_status: 'PROCESSING' })
    .eq('id', documentId);
  
  try {
    // 3. Get signed URL for document
    const { data: urlData } = await supabase.storage
      .from('documents')
      .createSignedUrl(doc.storage_path, 3600);
    
    // 4. Call Azure Document Intelligence (via n8n or direct)
    const ocrResult = await callAzureOCR(urlData.signedUrl);
    
    // 5. Store extraction result
    const { data: extraction } = await supabase
      .from('ocr_extractions')
      .insert({
        document_id: documentId,
        raw_text: ocrResult.rawText,
        extracted_fields: ocrResult.extractedFields,
        confidence_scores: ocrResult.confidenceScores,
        azure_operation_id: ocrResult.azureOperationId,
        model_version: ocrResult.modelVersion,
        processing_time_ms: ocrResult.processingTimeMs,
      })
      .select()
      .single();
    
    // 6. Run matching engine (only if not already assigned)
    if (!doc.claim_id) {
      const matchResult = await findMatchingClaim(
        ocrResult,
        doc.tenant_id
      );
      
      // 7. Store match decision
      await supabase
        .from('match_decisions')
        .insert({
          document_id: documentId,
          suggested_claim_id: matchResult.bestMatch?.claimId,
          match_score: matchResult.bestMatch?.score || 0,
          match_reasons: matchResult.bestMatch?.matchReasons || [],
          status: matchResult.assignmentType === 'AUTO' ? 'ACCEPTED' : 'PENDING',
          final_claim_id: matchResult.assignmentType === 'AUTO' 
            ? matchResult.bestMatch?.claimId 
            : null,
        });
      
      // 8. Auto-assign if above threshold
      if (matchResult.assignmentType === 'AUTO' && matchResult.bestMatch) {
        await supabase
          .from('documents')
          .update({
            claim_id: matchResult.bestMatch.claimId,
            match_status: 'AUTO',
            match_confidence: matchResult.bestMatch.score,
            folder_id: await determineFolder(
              matchResult.bestMatch.claimId,
              ocrResult.documentType
            ),
          })
          .eq('id', documentId);
        
        // Create timeline event
        await supabase
          .from('timeline_events')
          .insert({
            claim_id: matchResult.bestMatch.claimId,
            event_type: 'DOC_UPLOAD',
            title: `Dokument automatisch zugeordnet: ${doc.filename}`,
            metadata: {
              document_id: documentId,
              match_score: matchResult.bestMatch.score,
              match_reasons: matchResult.bestMatch.matchReasons,
            },
          });
      }
    }
    
    // 9. Update document with results
    await supabase
      .from('documents')
      .update({
        ocr_status: 'DONE',
        ocr_extraction_id: extraction.id,
        doc_type: ocrResult.documentType,
      })
      .eq('id', documentId);
    
    return new Response(JSON.stringify({ success: true }), {
      status: 200,
    });
    
  } catch (error) {
    // Mark as failed
    await supabase
      .from('documents')
      .update({ ocr_status: 'FAILED' })
      .eq('id', documentId);
    
    return new Response(JSON.stringify({ error: error.message }), {
      status: 500,
    });
  }
});
```

```typescript
  [DocumentType.ABNAHMEPROTOKOLL]: { folderType: 'PHASE', phaseAffinity: ClaimPhase.WH },
  
  [DocumentType.ANGEBOT]: { folderType: 'PHASE' }, // Determined by current phase
  [DocumentType.RECHNUNG]: { folderType: 'PHASE' }, // Determined by current phase
  
  [DocumentType.GUTACHTEN]: { folderType: 'GUTACHTEN' },
  [DocumentType.KORRESPONDENZ]: { folderType: 'KORRESPONDENZ' },
  [DocumentType.FOTO]: { folderType: 'PHASE' }, // Subfolder
  
  [DocumentType.SONSTIGES]: { folderType: 'KORRESPONDENZ' },
};

export async function determineFolder(
  claimId: string,
  docType: DocumentType,
  currentPhase?: ClaimPhase
): Promise<string> {
  const mapping = DOC_TYPE_FOLDER_MAP[docType];
  
  // Get claim's folders
  const folders = await getFoldersForClaim(claimId);
  
  if (mapping.folderType === 'PHASE') {
    const targetPhase = mapping.phaseAffinity || currentPhase;
    const folder = folders.find(
      f => f.folder_type === 'PHASE' && f.phase_affinity === targetPhase
    );
    return folder?.id || folders.find(f => f.folder_type === 'KORRESPONDENZ')!.id;
  }
  
  const folder = folders.find(f => f.folder_type === mapping.folderType);
  return folder?.id || folders[0].id;
}
```

```typescript
interface NotificationPayload {

  recipientIds: string[];
  claimId?: string;
  title: string;
  body: string;
  metadata?: Record<string, any>;
  actionUrl?: string;

export async function sendNotification(payload: NotificationPayload) {
  for (const recipientId of payload.recipientIds) {
    // Get user preferences
    const prefs = await getNotificationPreferences(recipientId);
    
    // In-App Notification (always check preference)
    if (prefs.get(payload.eventType, 'in_app')?.enabled) {
      await createInAppNotification({
        user_id: recipientId,
        event_type: payload.eventType,
        claim_id: payload.claimId,
        title: payload.title,
        body: payload.body,
        action_url: payload.actionUrl,
        metadata: payload.metadata,
      });
    }
    
    // Email Notification
    const emailPref = prefs.get(payload.eventType, 'email');
    if (emailPref?.enabled) {
      if (emailPref.frequency === 'immediate') {
        await queueEmail({
          to: await getUserEmail(recipientId),
          template: 'notification',
          data: payload,
        });
      } else {
        // Add to digest queue
        await addToDigestQueue(recipientId, payload, emailPref.frequency);
      }
    }
  }
}
```

```typescript
      const searchTerm = value.slice(lastAtIndex + 1);

        setShowMentionPopup(true);

        setShowMentionPopup(false);
      }

    setContent(newContent);
    setShowMentionPopup(false);

    await onSubmit({
      content,
      mentions,
      attachments: attachmentIds,
    });
    
    setContent('');
    setAttachments([]);
  };
  
  return (
```

```typescript
// middleware/document-access.ts

export async function checkDocumentAccess(
  documentId: string,
  userId: string,
  action: 'VIEW' | 'DOWNLOAD' | 'DELETE'
): Promise<boolean> {
  const user = await getUser(userId);
  const document = await getDocument(documentId);
  
  if (!document) return false;
  
  // System Admin: full access
  if (user.role === 'SYSTEM_ADMIN') return true;
  
  // Integration Account: full access
  if (user.role === 'INTEGRATION') return true;
  
  // Get claim for document
  const claim = document.claim_id 
    ? await getClaim(document.claim_id)
    : null;
  
  // Tenant match
  if (claim?.tenant_id === user.tenant_id) {
    // Tenant Admin: full access
    if (user.role === 'TENANT_ADMIN') return true;
    
    // Member: view/download only
    if (user.role === 'MEMBER') {
      return action !== 'DELETE';
    }
    
    // Viewer: view only
    if (user.role === 'VIEWER') {
      return action === 'VIEW';
    }
  }
  
  // Reviewer: check assignment
  if (user.role === 'REVIEWER' && claim) {
    const isAssigned = await isReviewerAssignedToClaim(userId, claim.id);
    if (isAssigned) {
      return action !== 'DELETE';
    }
  }
  
  return false;
}
```

```sql
-- Automatic audit trigger factory
CREATE OR REPLACE FUNCTION create_audit_trigger(table_name TEXT)
RETURNS VOID AS $$
BEGIN
    EXECUTE format('
        CREATE TRIGGER audit_%I
        AFTER INSERT OR UPDATE OR DELETE ON %I
        FOR EACH ROW EXECUTE FUNCTION log_audit_event()
    ', table_name, table_name);
END;
$$ LANGUAGE plpgsql;

-- Generic audit function
CREATE OR REPLACE FUNCTION log_audit_event()
RETURNS TRIGGER AS $$
BEGIN
    INSERT INTO audit_log (
        action,
        entity_type,
        entity_id,
        actor_id,
        actor_type,
        old_values,
        new_values
    ) VALUES (
        TG_OP,
        TG_TABLE_NAME,
        COALESCE(NEW.id, OLD.id),
        auth.uid(),
        CASE WHEN auth.uid() IS NULL THEN 'SYSTEM' ELSE 'USER' END,
        CASE WHEN TG_OP IN ('UPDATE', 'DELETE') THEN to_jsonb(OLD) END,
        CASE WHEN TG_OP IN ('INSERT', 'UPDATE') THEN to_jsonb(NEW) END
    );
    RETURN COALESCE(NEW, OLD);
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;
```

```sql
-- KPI-01 & KPI-02: Durchlaufzeiten
CREATE VIEW v_claim_durations AS
SELECT 
    c.id as claim_id,
    c.tenant_id,
    c.created_at,
    c.completed_at,
    EXTRACT(EPOCH FROM (c.completed_at - c.created_at)) / 86400 as total_duration_days,
    json_agg(json_build_object(
        'phase', p.phase_type,
        'duration_days', EXTRACT(EPOCH FROM (p.completed_at - p.started_at)) / 86400,
        'sla_met', p.completed_at <= p.sla_deadline
    )) as phase_durations
FROM claims c
LEFT JOIN phases p ON p.claim_id = c.id
WHERE c.completed_at IS NOT NULL
GROUP BY c.id;

-- KPI-03: SLA Quote
CREATE VIEW v_sla_performance AS
SELECT 
    tenant_id,
    phase_type,
    COUNT(*) as total_phases,
    COUNT(*) FILTER (WHERE completed_at <= sla_deadline) as on_time,
    ROUND(
        COUNT(*) FILTER (WHERE completed_at <= sla_deadline)::NUMERIC / 
        COUNT(*)::NUMERIC * 100, 
        2
    ) as sla_quote_percent
FROM phases
WHERE completed_at IS NOT NULL
GROUP BY tenant_id, phase_type;

-- KPI-04 & KPI-05: Schadenvolumen
CREATE VIEW v_claim_financials AS
SELECT 
    c.id as claim_id,
    c.tenant_id,
    c.erp_project_number,
    SUM(i.amount_net) as total_net,
    SUM(i.amount_gross) as total_gross,
    COUNT(i.id) as invoice_count
FROM claims c
LEFT JOIN invoices i ON i.claim_id = c.id AND i.type = 'INVOICE'
GROUP BY c.id;

    SELECT

        c.tenant_id,
        calculate_traffic_light(c.id) as traffic_light
    FROM claims c
    WHERE c.completed_at IS NULL
)
SELECT 
    tenant_id,
    traffic_light,
    COUNT(*) as count
FROM claim_traffic_lights
GROUP BY tenant_id, traffic_light;

CREATE VIEW v_ocr_performance AS
SELECT 
    d.tenant_id,
    DATE_TRUNC('month', d.uploaded_at) as month,
    COUNT(*) as total_documents,
    COUNT(*) FILTER (WHERE d.match_status = 'AUTO') as auto_assigned,

    COUNT(*) FILTER (WHERE d.match_status = 'MANUAL') as manually_assigned,

    ROUND(
        COUNT(*) FILTER (WHERE d.match_status = 'AUTO')::NUMERIC / 
        NULLIF(COUNT(*), 0)::NUMERIC * 100,

    ) as auto_assignment_rate_percent
```

```typescript
        durations: {
          avgTotalDays: avgDurations.total,
          byPhase: avgDurations.byPhase,
          p50: avgDurations.p50,
          p90: avgDurations.p90,
        },
        financials: {
          totalVolume: financials.totalNet,
          avgClaimValue: financials.avgPerClaim,
          openInvoices: financials.openInvoices,
        },
        ocr: {
          autoAssignmentRate: ocrPerformance.autoRate,
          reviewQueueSize: ocrPerformance.reviewQueue,
        },
      };
    }),
  
  exportReport: protectedProcedure
    .input(z.object({
      tenantId: z.string().uuid(),
      reportType: z.enum(['CLAIMS', 'FINANCIALS', 'DURATIONS']),
      format: z.enum(['CSV', 'XLSX']),
      dateFrom: z.date(),
      dateTo: z.date(),
    }))
    .mutation(async ({ input, ctx }) => {
      // Check export permission
      await assertPermission(ctx.user, 'stats:export');
      
      const data = await generateReportData(input);
      const file = await formatReport(data, input.format);
      
      return {
        downloadUrl: await uploadTempFile(file),
        expiresAt: addHours(new Date(), 24),
      };
    }),
});
```

## M) Offene Fragen / Annahmen / Risiken (ADR-Style)

### ADR-001: ERP API Specification

| | |
|---|---|
| **Status** | ⚠️ OFFEN |
| **Kontext** | Das Framework basiert auf der Annahme einer REST/Webhook-basierten ERP-API. Die exakte Spezifikation ist nicht bekannt. |
| **Annahmen** | - ERP kann Webhooks senden bei Status-Änderungen<br>- ERP hat REST-Endpoints für Dokument-Upload<br>- ERP kann Projektnummer als Identifier verwenden |
| **Risiko** | Falls ERP keine Webhooks unterstützt: Polling nötig (Latenz, Ressourcen)<br>Falls ERP-API nicht dokumentiert: Reverse Engineering nötig |
| **Aktion** | ERP-Team kontaktieren für API-Dokumentation vor Phase 2 Start |

### ADR-002: OCR Model Training

| | |
|---|---|
| **Status** | ⚠️ ENTSCHEIDUNG NÖTIG |
| **Kontext** | Azure Document Intelligence bietet Prebuilt-Modelle. Für höhere Accuracy bei domänenspezifischen Dokumenten (LO-Protokoll, Angebot) könnten Custom Models nötig sein. |
| **Option A** | Prebuilt Models only (schneller, günstiger, ~75-80% Accuracy) |
| **Option B** | Custom Models für Top-5-Dokumenttypen (Aufwand: ~2 Wochen, ~90%+ Accuracy) |
| **Empfehlung** | Start mit Prebuilt (Phase 2), Custom Models als Phase 3 Feature |

### ADR-003: Tenant Onboarding

| | |
|---|---|
| **Status** | ⚠️ OFFEN |
| **Kontext** | MVP sieht manuelle Tenant-Erstellung vor. Skalierung erfordert Self-Service. |
| **Fragen** | - Wer legt Tenants an? (SWS Admin vs. Self-Service)<br>- Wie erfolgt Billing? (pro User, pro Claim, Flat Fee)<br>- Braucht es Trial-Period? |
| **Annahme** | Phase 1: SWS Admin legt Tenants manuell an. Self-Service in Phase 3. |

### ADR-004: Document Storage Location

| | |
|---|---|
| **Status** | ✅ ENTSCHIEDEN |
| **Kontext** | Dokumente können in Supabase Storage oder ERP Storage liegen. |
| **Entscheidung** | Supabase Storage als Primary für Portal-Uploads. ERP-Dokumente werden referenziert (URL), nicht kopiert. |
| **Begründung** | - Vermeidet Storage-Duplikation<br>- ERP bleibt Source of Truth<br>- Portal kann Thumbnails/Preview cachen |

### ADR-005: Reviewer Role Cross-Tenant

| | |
|---|---|
| **Status** | ⚠️ RISIKO |
| **Kontext** | Sachverständige/Regulierer benötigen Zugriff auf Schäden mehrerer Tenants. |
| **Risiko** | RLS-Komplexität steigt. Potenzielle Security Vulnerabilities bei Fehlkonfiguration. |
| **Mitigation** | - Explizite `claim_reviewers` Junction Table<br>- Quarterly Security Audit für Cross-Tenant Access<br>- Logging aller Cross-Tenant Zugriffe |

### ADR-006: Mobile App Strategy

| | |
|---|---|
| **Status** | ✅ ENTSCHIEDEN |
| **Options** | Native iOS/Android vs. React Native vs. PWA |
| **Entscheidung** | PWA first (Phase 1+2). React Native nur bei nachgewiesenem Bedarf (Push Notifications, Offline-First). |
| **Begründung** | - Schnellere Time-to-Market<br>- Einheitliche Codebase<br>- Zielgruppe nutzt primär Desktop |

### ADR-007: Traffic Light Persistence

| | |
|---|---|
| **Status** | ⚠️ ENTSCHEIDUNG NÖTIG |
| **Kontext** | Traffic Light kann live berechnet oder materialisiert (DB) werden. |
| **Option A** | Live Calculation (immer aktuell, aber Latenz bei vielen Claims) |
| **Option B** | Materialized (schnell, aber Sync-Delay) |
| **Empfehlung** | Hybrid: Live für Detailseite, Materialized View für Map/Liste (5-min Refresh) |

### ADR-008: Notification Delivery Guarantee

| | |
|---|---|
| **Status** | ⚠️ OFFEN |
| **Kontext** | In-App Notifications sind einfach, E-Mail erfordert zuverlässige Delivery. |
| **Fragen** | - Welcher E-Mail Provider? (Resend, SendGrid, SES)<br>- Bounce/Complaint Handling?<br>- DKIM/SPF Setup für sws-fm.de Domain? |
| **Annahme** | Phase 1: Resend (einfach, günstig). Migration zu Enterprise Provider in Phase 3. |
